# SuperChat Messenger

Консольный P2P мессенджер с E2EE (сквозным шифрованием) на базе Python, Textual и SQLite.
Мессенджер работает по гибридной архитектуре: клиенты общаются напрямую друг с другом по IP (в локальной сети или через прямой интернет), а если P2P-соединение невозможно — обмениваются сообщениями через центральный релей-сервер.

## 🚀 Установка для пользователей

### 1. Как запустить Сервер (на выделенном хосте/ПК)
Вам не нужно клонировать репозиторий, если у вас установлен **Docker**. Сервер опубликован в реестре GitHub. Достаточно выполнить одну команду:

```bash
docker run -d -p 8888:8888 --name superchat-server ghcr.io/art9762/superchat:latest
```
*(Убедитесь, что порт 8888 открыт в вашем файрволе для входящих TCP-соединений).*

---

### 2. Как запустить Клиент (UI)
Клиенту не нужен Docker. Для работы вам потребуется Python версии 3.10 или выше.

1. Склонируйте репозиторий на свой компьютер:
   ```bash
   git clone https://github.com/art9762/SuperChat.git
   cd SuperChat
   ```
2. Создайте виртуальное окружение и установите зависимости:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r client/requirements.txt
   ```
3. Запустите мессенджер:
   ```bash
   python3 client/ui.py
   ```

*Примечание: По умолчанию клиент подключается к серверу `127.0.0.1:8888`. Если ваш сервер находится в сети интернет, отредактируйте `client/ui.py` (строку `self.server_host = "127.0.0.1"`) на IP-адрес вашего сервера.*

## 🏗️ Разработка и локальная сборка
Если вы хотите собрать Docker образ сервера самостоятельно (из исходников), мы предусмотрели скрипты в `Makefile`. Так как образ мульти-архитектурный, вы можете собрать его под свой ПК или сервер:

- Для сборки образа под сервер Linux/Ubuntu (x86_64 / amd64):
  ```bash
  make build-amd64
  ```
- Для сборки образа под Apple Silicon Mac / Raspberry Pi (arm64):
  ```bash
  make build-arm64
