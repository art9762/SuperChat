# SuperChat — описание проекта

## Что это

SuperChat — консольный мессенджер со сквозным шифрованием (E2EE), написанный на Python. Работает полностью в терминале: никакого браузера, никакого GUI. Интерфейс построен на библиотеке urwid — Python-аналоге React+Ink, на котором сделан Claude Code.

Основная идея: два человека открывают терминал, запускают `superchat`, и общаются напрямую — сервер не видит содержимого сообщений, даже если оба подключены через него.

---

## Архитектура

Проект состоит из двух независимых компонентов: **сервера** и **клиента**.

### Гибридная доставка сообщений

SuperChat использует два канала доставки и выбирает лучший автоматически:

```
┌──────────┐   P2P (прямое соединение)   ┌──────────┐
│  Клиент  │ ─────────────────────────→  │  Клиент  │
│   Alice  │                             │   Bob    │
└──────────┘                             └──────────┘
      │                                        │
      │   relay (если P2P недоступен)          │
      └──────────────┬─────────────────────────┘
                     │
               ┌─────▼─────┐
               │  Сервер   │
               │  (relay)  │
               └───────────┘
```

**Приоритет 1 — P2P:** клиент пробует подключиться к собеседнику напрямую по его IP:порт. Это работает, если оба в одной сети или у собеседника открытый порт.

**Приоритет 2 — Relay:** если P2P не удался, сообщение отправляется через центральный сервер. Сервер выступает только как слепой ретранслятор — он видит отправителя и получателя, но не содержимое (оно зашифровано).

## Шифрование

Используется **гибридная схема RSA-2048 + AES (Fernet)** — та же схема, что в PGP и Signal.

### Почему не только RSA

RSA-2048 может зашифровать не более ~190 байт за раз. Для сообщений и тем более файлов это не подходит. Поэтому:

```
Отправитель:
  1. Генерирует случайный AES-ключ (Fernet, 256 бит)
  2. Шифрует текст/файл этим AES-ключом  →  зашифрованные данные
  3. Шифрует AES-ключ публичным RSA-ключом получателя  →  зашифрованный ключ
  4. Склеивает: [4 байта длины ключа] + [зашифрованный ключ] + [зашифрованные данные]
  5. Кодирует в Base64 и отправляет

Получатель:
  1. Декодирует Base64
  2. Читает длину ключа из первых 4 байт
  3. Расшифровывает AES-ключ своим приватным RSA-ключом
  4. Расшифровывает данные извлечённым AES-ключом
```

### Ключи

При первом запуске клиент генерирует пару RSA-ключей и сохраняет `private.pem` локально. Публичный ключ передаётся на сервер при регистрации. Сервер рассылает публичные ключи всем участникам, шифруя каждую рассылку ключом соответствующего получателя (RSA + Fernet). Таким образом, даже список онлайн-пользователей передаётся в зашифрованном виде.

## Сервер

**Файл:** `server/server.py`

Сервер — минималистичный asyncio TCP-сервер, не хранящий никаких данных на диске. В памяти держит словарь `username → ConnectedClient` с writer, IP, P2P-портом и публичным ключом.

**Функции:**
- `register` — принимает регистрацию клиента, обновляет список онлайна, рассылает `users_update` всем
- `relay` — пересылает зашифрованный payload от A к B как `relayed_message`
- При отключении клиента автоматически рассылает обновлённый список онлайна

Сервер **не может** читать сообщения: он работает только с зашифрованными Base64-блобами.

Разворачивается через Docker:
```bash
docker run -d -p 8888:8888 ghcr.io/art9762/superchat:latest
```
Docker-образ мультиархитектурный: собирается под linux/amd64 и linux/arm64.

## Клиент

### Структура клиента

```
client/
├── ui.py       — интерфейс (urwid + asyncio)
├── network.py  — сетевой движок (P2P + relay, шифрование при отправке)
├── crypto.py   — RSA/AES шифрование и управление ключами
├── db.py       — SQLite: контакты, история сообщений, настройки
└── private.pem — RSA приватный ключ (генерируется при первом запуске)
```

### Интерфейс (ui.py)

Построен на **urwid** — TUI-фреймворке с компонентной моделью и asyncio event loop. Urwid перерисовывает только изменившиеся части экрана без мерцания — то же, что делает React+Ink в Claude Code.

Структура экрана:

```
 ◆ superchat  ·  artem  ● connected       ← header bar
──────────────────────────────────────────
  alice  14:32
  hey, want to grab lunch?

  artem  14:33
  sure!

  ╭─ file received ──────────────────────╮
  │  photo.png · 2.3 MB                  │
  ╰──────────────────────────────────────╯

──────────────────────────────────────────
 alice › _                                ← input
```

Команды в интерфейсе:

| Команда | Действие |
|---|---|
| `/chat <user>` | Открыть чат с пользователем |
| `/contacts` | Список онлайн-контактов |
| `/history` | Загрузить историю текущего чата |
| `/send <path>` | Отправить файл текущему собеседнику |
| `/back` | Закрыть текущий чат |
| `/help` | Показать список команд |
| `/quit` | Выйти |

### Сетевой движок (network.py)

- При старте запускает локальный P2P TCP-сервер на случайном порту
- Подключается к центральному серверу, регистрируется (имя + P2P-порт + публичный ключ)
- При отправке: пробует P2P → при неудаче отправляет через relay
- При получении: расшифровывает payload, определяет тип (текст или файл)
- Файлы сохраняются в `~/.superchat/downloads/`
- При разрыве соединения с сервером — переподключается каждые 5 секунд

### База данных (db.py)

SQLite-файл `history.db` хранит три таблицы:

- `contacts` — известные пользователи: имя, публичный ключ, последние IP и порт
- `messages` — история переписки: контакт, отправитель, текст, время
- `settings` — настройки клиента (имя пользователя, и т.д.)

## Поток данных: отправка сообщения

```
Пользователь вводит текст
        │
        ▼
ui.py: _dispatch()
        │
        ▼
network.py: send_message()
        │
        ├──► Получить публичный ключ получателя из SQLite
        │
        ├──► crypto.encrypt_for(public_key, text)
        │         └── генерировать AES-ключ
        │         └── зашифровать текст AES
        │         └── зашифровать AES-ключ RSA-ключом получателя
        │         └── вернуть Base64(len + enc_key + enc_data)
        │
        ├──► Попытка P2P: open_connection(ip, p2p_port)
        │         └── успех → отправить {"type":"p2p_message", "payload": ...}
        │
        └──► Fallback Relay: server_writer.write({"type":"relay", "to":..., "payload":...})
```

## Установка

### Сервер

```bash
docker run -d -p 8888:8888 --name superchat-server ghcr.io/art9762/superchat:latest
```

### Клиент (macOS/Linux)

```bash
curl -sSL https://raw.githubusercontent.com/art9762/SuperChat/main/install.sh | bash
superchat
```

### Клиент (Windows, PowerShell)

```powershell
iwr -useb https://raw.githubusercontent.com/art9762/SuperChat/main/install.ps1 | iex
superchat
```

Скрипт клонирует репозиторий в `~/.superchat`, создаёт virtualenv, устанавливает зависимости и прописывает команду `superchat` в `~/.local/bin`.
